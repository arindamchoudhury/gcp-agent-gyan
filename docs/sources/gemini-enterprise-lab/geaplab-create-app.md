## Summary

Hands-on walkthrough of creating a Gemini Enterprise app from scratch: provisioning the app, connecting three data stores, and exploring four major capabilities — general AI assistant, multimodal search, pre-built agents (Idea Generation and Deep Research), and NotebookLM Studio.

## Key points

- App creation takes four steps: name the app → set up identity (Google Identity as workforce provider) → connect data stores → share URL with workforce.
- Three data store types demonstrated: Google Drive (entire org shared drives), Google Calendar (named connector), Google Cloud Storage (unstructured documents — PDFs, HTML, TXT; one-time sync from a bucket).
- The "enhance your conversation" toggle lets users scope search to internal sources only and enables multimodal prompts (e.g., "Summarize the customer sentiment pie chart").
- Two featured pre-built agents: **Idea Generation** (team of agents plan → generate → evaluate) and **Deep Research** (research plan → user review/modify → comprehensive report + audio summary).
- NotebookLM Studio transforms source material into dynamic formats: audio, video overviews, mind maps, reports — goes beyond Q&A into content creation.

### App creation workflow

From Google Cloud Console:
1. Search "Gemini Enterprise" → select product → **Create your first app**
2. Name the app (e.g., "Symbol Foods Marketing Team") → **Create**
3. **Setup identity** → choose Google Identity → **Confirm Workforce Identity**
4. **Connected data stores** → add data stores (see below)
5. Share the generated unique URL with the workforce

### Connecting data stores

From the Connected data stores page → **New data store**:

| Data store | Configuration |
|---|---|
| **Google Drive** | Sync all organization's shared drives → Continue → name it → Create |
| **Google Calendar** | Select connector → name it → Create |
| **Google Cloud Storage** | Data type: Documents (unstructured: PDFs, HTML, TXT) · Sync frequency: One time · Browse to select bucket → Continue → name it → Create |

This confirms the enterprise data hub role described in [[gemini-enterprise-intro]]: Google Drive, Calendar, and GCS are all first-class data store connectors.

### General assistant and multimodal search

- Prompt in the homepage box → Gemini searches internal and external data sources → context-aware response.
- **Enhance your conversation** button → scope to internal sources only.
- Multimodal prompt example: "Summarize the customer sentiment pie chart" → Gemini finds the chart image in internal sources and generates a summary.

### Pre-built agents in practice

This walkthrough puts flesh on the "pre-built agents" category described in [[gemini-enterprise-intro]]:

**Idea Generation**
> "A team of agents to plan, generate, and evaluate ideas on your topic."
- Enter prompt (e.g., "brainstorm a marketing campaign for Gen-Z customers") → generates plan → review → **Start session** → multiple ideas generated by the agent team.

**Deep Research**
> "Performing in-depth analysis and generating comprehensive reports on complex topics."
- Enter research question → Gemini drafts a research plan → review/modify → **Start research** → comprehensive report + audio summary generated.

Both follow the same pattern: agent proposes a plan for review before executing — keeping the human in the loop before the expensive compute step.

### NotebookLM in practice

Sources can be: Google Docs, website URLs, YouTube videos, copied text, or uploaded files. Once loaded:
- Chat-style Q&A with citations against the uploaded corpus.
- **Save responses as notes** for later retrieval.
- **Studio tab**: transform source material into audio overviews, video presentations ("Marketing by the Numbers"), mind maps, and reports.

This extends [[gemini-enterprise-intro]]'s description of NotebookLM as "in-depth research from provided sources" — Studio shows it's also a *content creation* tool, not just a research tool.
